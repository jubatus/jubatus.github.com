
How to Contribute to Jubatus
============================

First of all please subsribe to the `mailing list <http://groups.google.com/group/jubatus>`_ to reach developers' and users' community. Currently we don't separate dev ML and user ML because Jubatus has just started now. Please,

- Give us a feedback or tell us how do you use Jubatus in your environment via the `mailing list <http://groups.google.com/group/jubatus>`_ .
- Submit a bugreport to `github issues <https://github.com/jubatus/jubatus/issues>`_ .
- Improve documentation or Suggest new feature by sending `pull request <https://github.com/jubatus/website/pulls>`_ .
- Fix `bugs <https://github.com/jubatus/jubatus/issues>`_ .


Repositories
------------

- `Jubatus <http://github.com/jubatus/jubatus>`_

 - Jubatus software including classifier, recommender, stat, recommender and server framework.
 - branches - Committers use `git-flow <https://github.com/nvie/gitflow>`_ under `A successful Git branching model <http://nvie.com/posts/a-successful-git-branching-model/>`_ .

  - develop - Development branch. Latest features are pushed here.
  - master  - Release branch.

- `Website <http://github.com/jubatus/website>`_

 - `Sphinx <http://sphinx.pocoo.org/>`_ source of website (this site). Every contribution such as typo fix, English grammer fix will be welcomed.
