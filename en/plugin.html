Plugin Develpment¶

Jubatus can extend its functionality with user-developed plugins. Especially in data conversion engine, we can develop plugins for each module so that we can embed arbitrary conversion logics. Plugin is written in C++. In this section, we will explain how to develop plugins.

Plugin for Data Conversion¶

Developing plugin is roughly developed in the following manner. First, we include /usr/local/include/jubatus/plugin.hpp. Second, we inherit a base class, or a framework of plugin and write an arbitrary module for feature extraction and filter. Finally, we write funcion that creates an instance of the class and returns a pointer to the instance. This function must be enclosed with extrn “C” so that it is seen from C code. The name of the function is specified as “function” key in fv_converter when using this plugin.

As an example, we describe how to develop a plugin for “string_types”. In this case, base class to be inherit is jubatus::word_splitter class. word_splitter class is consist of the only member function named “split”. split function receives string, splits this string and returns split point. The following is an example of my_splitter class.

#include <jubatus/plugin.hpp>
#include <map>

using namespace std;

class my_splitter : public jubatus::word_splitter {
 public:
   void split(const string& string,
              vector<pair<size_t, size_t> >& ret_boundaries) {
    // do somehting
  }
};

extern "C" {
  my_splitter* create(const map<string, string>& params) {
    return new my_splitter();
  }
}

We compile this file and like it with libjubaconverter.so and get .so file. We can use this .so file as a plugin. How to use plugins is described in Data Conversion and Plugins.

We can develop plugins of other feature extractors and filter in the same way. src/server/fv_converter/test_*.cpp are sample of each type of plugin. So please refer to these file for more concrete explanation.
